# Governance

## Project Maintainer
This project is currently maintained by a single maintainer. The maintainer is responsible for:
- Reviewing and merging pull requests
- Managing releases
- Responding to security reports
- Setting project direction
- Enforcing the Code of Conduct


## Decision Making
Decisions are made through consensus. When consensus cannot be reached, the maintainer makes the final decision.

## Becoming a Maintainer
Regular contributors may be invited to become maintainers. Criteria include:
- Consistent, high-quality contributions
- Understanding of the project architecture
- Active participation in discussions and reviews
- Demonstration of good judgment and community values

## Contributor Roles

### First-time Contributor
- Makes an initial contribution
- Completes the CLA if required


### Repeat Contributor
- Makes multiple contributions
- Familiar with project conventions


### Core Contributor
- Regular contributor with deep project knowledge
- May be asked to review PRs
- May be invited to become a maintainer


## Community
- **Issues**: Bug reports and feature requests
- **Discussions**: Questions, ideas, community support
- **Pull Requests**: Code contributions


## Code of Conduct Enforcement
Reports of Code of Conduct violations are handled by the maintainer. All reports are kept confidential.
